Ajax

  • Web 1.O: contenu satique, on recharge la page pour présenter de nouvelles données
  • Arrive le XMLHttpRequest (basé sur des travaux de Microsoft pour ActiveX pour Outlook WA)
  • Ajax combine XMLHttpRequest, JavaScript, JSON, XML, DOM HTML et css pour dynamiser l'application
  • On parle alors d'Application Internet Riche (RIA)

Ajax permet d'échanger des données entre le client et le serveur sans devoir recharger la page.

Malheureusement l'api est implémentée différament par chaque navigateur ce qui complique son utilisation.

jQuery

=> C'est une librairie JavaScript.

=> Elle permet de simplifier les appels Ajax et la manipulation du DOM.

=> Extensible avec des plugins.

  • HTML/DOM manipulation
  • CSS manipulation
  • HTML event methods
  • Effects and animations
  • AJAX

jQuery object methods: $.()

Au chargement du DOM: $(document).ready()

Séléction

  • Par ID: $("#myId");
  • Par classe css: $(".myClass");
  • Par attribut: $("input[name='first_name']");
  • Par composition: $("#contents ul.people li");

Manipulation d'élément

  • .attr()
  • .html();
  • .text();
  • .val();
  • .css();
  • .add/remove/toggleClass()

jQuery Ajax

  • GET / POST
  • Asynchrone
  • text / html / script / json / jsonp / xml

La méthode principale: $.ajax()

$.ajax({
    url: "post.php",
    data: { id: 123 },
    type: "GET",
    dataType : "json",
})
  .done(function( json ) {})
  .fail(function( xhr, status, errorThrown ) {})
  .always(function( xhr, status ) {});

Shorthand $.get/$.post/$.getScript/$.getJSON

$.get( "/users.php", { userId: 1234 }, function( resp ) {} );

Une fonction particulière: $().load("url/fichier.html") charge le contenu de l'url (html) dans l'élément.
